# 다재다능 스택 측정기

리그 오브 레전드 **다재다능(Jack of All Trades)** 룬의 Jack 스택을 정확히 계산하는 도구.
소환사 협곡 아이템을 골라 빌드를 짜면 *서로 다른 스탯 종류*의 개수로 스택과 보너스를 보여준다.

## 실행

- **가장 간단:** `index.html`을 더블클릭 (브라우저에서 DDragon·Community Dragon 최신 데이터를 직접 받아옴, 인터넷 필요)
- **로컬 서버로:** `node serve.mjs` → http://localhost:5544
  - Claude Code 프리뷰: `.claude/launch.json`의 `jack-tool` 구성 사용

## 정확성 설계 (기존 사이트가 틀리는 부분)

1. **`<stats>` 블록을 직접 파싱** — DDragon의 구조화된 `stats` 필드는 *치명타 피해량* 등 최신 스탯이 누락돼 있어, description의 `<stats>...</stats>`에서 기본 스탯을 직접 추출한다.
2. **협곡 빌드 아이템만** — `maps[11] && gold.purchasable`로 아레나/칼바람 동명 중복을 거르고, 추가로 **소모품(물약·영약·와드)·장신구·숨김(`hideFromAll`)·챔피언 전용(`requiredChampion`)을 제외**한다. 영약 등 버프 스탯은 룬에서 애초에 인정되지 않으므로 빼는 것이 정확하다.
3. **중복 제거** — 같은 이름이 두 번 나오는 모드 변형본(`32xxxx`/`66xxxx` id)·정글펫 쌍은 **가장 낮은 id(표준 원본)만 남기고** 제거한다.
4. **아레나·칼바람·삭제템 등 협곡 외 아이템 제외** — DDragon `maps` 플래그는 부정확하다(아레나·칼바람 아이템에도 `maps[11]`이 켜져 있음). 두 단계로 거른다: **①** 이름별 최소 id(기본형) 엔트리가 협곡(11)일 때만 인정(아레나=기본형 맵 30 등, 삭제템=기본형 맵 빈 배열 제외). **②** **Community Dragon(게임 클라이언트 데이터)으로 교차검증** — `inStore=true`이면서 아이콘 경로에 `ARAM`이 없어야 인정(수호자의 망치 등 칼바람 전용 제거). 앱이 DDragon·CDragon을 함께 라이브로 받아 적용한다(둘 다 CORS `*`). 제외 예: 꽃피는 새벽의 검·신성의 검·아트마의 심판·가고일 돌갑옷·부서진 여왕의 왕관·수호자의 망치. (현재 **197개**) · "10초당 골드"는 현재 서포터 완성형 아이템의 정식 스탯이라 그대로 인정한다.
5. **고정 vs % 분리** — *이동 속도*와 *마법 관통력*은 고정값과 %가 **각각 다른 스택**으로 인정된다.
6. **적응형 능력치 제외** — `적응형 능력치`로 표기된 스탯은 Jack 스택을 주지 않는다.
7. **아이템 기본 스탯만** — 패시브/액티브로 주는 스탯(`<passive>`/`<active>`)은 세지 않는다.

## 룬 메커니즘 (현재 기준)

- 아이템에서 얻는 **서로 다른 스탯 1종류 = Jack 1스택** (같은 스탯은 여러 아이템에서 나와도 1스택)
- 스택당 **스킬 가속 +1** (최대 10)
- **5스택**: 적응형 +10 (공격력 6 또는 주문력 10)
- **10스택**: 적응형 +25 (공격력 15 또는 주문력 25)
- 스택 상한 **10**

## 데이터 갱신

새 패치가 나오면 앱은 자동으로 최신 데이터를 받지만, 오프라인 캐시/검사용 파일을 갱신하려면:

```bash
node build-data.mjs --live   # DDragon 최신 패치를 받아 raw + data/items.json 갱신 + 검증 출력
node build-data.mjs          # 오프라인(ddragon_item_raw.json)으로 재생성 + 검증
```

`build-data.mjs`는 매핑 실패 스탯, 인정 스탯별 아이템 수, 샘플 빌드 계산을 출력해 정확성을 점검한다.
새 패치에서 새로운 스탯 표기가 등장하면 "알 수 없는 스탯 표기"로 잡히며, `STAT_MAP`(index.html과 build-data.mjs 양쪽)에 추가하면 된다.

## 파일

| 파일 | 역할 |
|------|------|
| `index.html` | 단일 파일 앱 (파싱 로직 + 계산기 UI 포함) |
| `build-data.mjs` | 데이터 빌드 + 정확성 검증 (Node) |
| `serve.mjs` | 의존성 없는 정적 서버 |
| `data/items.json` | 생성된 협곡 아이템 데이터(검사용) |
| `ddragon_item_raw.json` | DDragon 원본 캐시(오프라인용) |
